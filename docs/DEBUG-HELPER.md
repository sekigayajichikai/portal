### 2. 重要な確認ポイント

#### A. データベースから取得したバス停の数
- **App.tsx - ユニークなバス停の数**: この数字が実際にデータベースに登録されているバス停の数です
- もし4件なら、データベースには4つのバス停しか登録されていません
- もし5件以上なら、次のステップへ

#### B. お気に入りに登録されているバス停
- **お気に入りバス停の件数**: この数字がお気に入りに登録されている数です
- もし1件登録されていて、全体が5件なら、ドロップダウンには4件表示されます

#### C. ドロップダウンに表示される件数
- **ドロップダウンに表示する件数**: これが実際にドロップダウンに表示される数です
- 計算式: `全バス停の数 - お気に入りの数 = ドロップダウンに表示される数`

### 3. ローカルストレージの確認

開発者ツールで以下を確認：

1. **Application** タブ（または **Storage** タブ）を開く
2. 左側のメニューから **Local Storage** を展開
3. サイトのURLを選択
4. `favoriteBusStops` というキーを探す
5. その値を確認（配列形式で保存されています）

### 4. ローカルストレージをリセット

お気に入りをリセットして確認したい場合：

#### 方法1: 開発者ツールから削除
1. Local Storageの `favoriteBusStops` を右クリック
2. **Delete** を選択
3. ページをリロード

#### 方法2: コンソールから削除
開発者ツールのコンソールで以下を実行：

```javascript
localStorage.removeItem('favoriteBusStops');
location.reload();
```

#### 方法3: すべてのローカルストレージをクリア
```javascript
localStorage.clear();
location.reload();
```

### 5. データベースの確認

Supabaseダッシュボードで以下のSQLを実行：

```sql
-- アクティブなバス時刻表の件数を確認
SELECT COUNT(*) FROM bus_schedules WHERE is_active = true;

-- ユニークなバス停名を確認
SELECT DISTINCT stop_name FROM bus_schedules WHERE is_active = true ORDER BY stop_name;

-- 全データを確認（詳細）
SELECT
  stop_name,
  destination,
  route_name,
  is_active
FROM bus_schedules
ORDER BY stop_name, destination;
```

## 問題のパターンと解決方法

### パターン1: データベースに4件しかない場合

**症状**:
- App.tsx のログで「ユニークなバス停の数: 4件」と表示される
- ドロップダウンに4件表示される

**原因**: データベースに実際に4つのバス停しか登録されていない

**解決方法**:
1. 管理画面でバス時刻表のPDFをアップロードして追加登録
2. またはSupabaseダッシュボードで直接レコードを追加

### パターン2: お気に入りに登録されているバス停が多い場合

**症状**:
- App.tsx のログで「ユニークなバス停の数: 5件以上」と表示される
- お気に入りの件数が1件以上
- ドロップダウンには `全体 - お気に入り = 4件` 表示される

**原因**: お気に入りに既に登録されているバス停があるため、ドロップダウンから除外されている

**解決方法**:
1. 上記の「ローカルストレージをリセット」を実行
2. ページをリロードして確認

### パターン3: データの取得に失敗している場合

**症状**:
- コンソールに「❌ バス時刻表の取得に失敗しました」というエラーが表示される

**原因**: Supabaseとの接続エラー

**解決方法**:
1. Supabaseの認証情報を確認
2. ネットワーク接続を確認
3. Supabaseのダッシュボードでテーブルが正しく作成されているか確認

## 次のステップ

コンソールログを確認して、上記のどのパターンに該当するか特定してください。
該当するパターンの解決方法を実行すれば、問題が解決するはずです。
