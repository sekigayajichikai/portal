# ラジオ回覧板生成機能 セットアップ＆テストガイド

## 概要

デジタル回覧板に登録された記事から、AIが2人のDJによる掛け合い形式のラジオ回覧板を自動生成する機能です。

## 機能の特徴

- **自動台本生成**: Gemini 2.0 FlashがPDFから抽出された記事を元に、2人のDJの掛け合い台本を生成
- **音声合成**: Gemini TTSで台本を音声化（3-5分の番組）
- **クラウド保存**: Supabase Storageに音声ファイルを保存
- **公開アプリで再生**: 一般ユーザーが生成されたラジオ回覧板を聴取可能

## セットアップ手順

### 1. データベースのセットアップ

Supabase Dashboard > SQL Editor で以下のSQLスクリプトを実行してください：

```bash
# radio_programs テーブルを作成
d:/ドキュメント/GitHub/CC-SaaS/radio-programs-setup.sql
```

**確認事項:**
- `radio_programs` テーブルが作成されたこと
- インデックスが正しく設定されていること

### 2. ストレージのセットアップ

#### ステップ2.1: バケットを作成

1. Supabase Dashboard > Storage に移動
2. 「Create bucket」をクリック
3. 以下の設定でバケットを作成：
   - **Bucket name**: `radio`
   - **Public bucket**: ✓ チェックを入れる（必須）
   - **File size limit**: 50MB（推奨）
4. 「Create bucket」をクリック

#### ステップ2.2: ポリシーを設定

Supabase Dashboard > SQL Editor で以下のSQLスクリプトを実行してください：

```bash
# ストレージポリシーを設定
d:/ドキュメント/GitHub/CC-SaaS/radio-storage-setup.sql
```

**確認事項:**
- `radio` バケットが存在すること
- バケットがPublicに設定されていること
- 3つのポリシーが適用されていること

### 3. 環境変数の確認

`.env` ファイルに以下の環境変数が設定されていることを確認：

```env
# 必須
VITE_GEMINI_API_KEY=your_gemini_api_key_here
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```

## 使い方

### 管理画面（Admin）でラジオ回覧板を生成

1. **管理画面を起動**
   ```bash
   npm run dev:admin
   ```

2. **デジタル回覧板を開く**
   - 「デジタル回覧板」タブ > 「保存済み一覧」を選択
   - 任意のデジタル回覧板をクリック

3. **ラジオ回覧板を生成**
   - 「ラジオ回覧板を生成」ボタンをクリック
   - 確認ダイアログで「OK」をクリック

4. **生成を待つ（約1-2分）**
   - 進捗バーで進行状況を確認
   - 生成が完了すると音声プレイヤーが表示されます

5. **プレビュー再生**
   - 生成された音声をその場で再生して確認
   - 問題があれば「ラジオを再生成」で再作成可能

### 一般ユーザーアプリ（Public）で再生

1. **一般ユーザーアプリを起動**
   ```bash
   npm run dev:public
   ```

2. **ラジオ画面を開く**
   - ボトムナビゲーションから「ラジオ回覧板」をタップ

3. **ラジオ回覧板を選択**
   - プレイリストから聴きたい回覧板をタップ

4. **再生**
   - 再生ボタン（▶️）をタップして回覧板を聴く

## トラブルシューティング

### 問題1: 「Gemini APIキーが設定されていません」

**原因**: 環境変数が正しく設定されていない

**解決方法**:
1. `.env` ファイルに `VITE_GEMINI_API_KEY` が設定されているか確認
2. 開発サーバーを再起動: `npm run dev`

### 問題2: 「音声ファイルのアップロードに失敗しました」

**原因**: Supabase Storageの設定が不完全

**解決方法**:
1. `radio` バケットが作成されているか確認
2. バケットが **Public** に設定されているか確認
3. ストレージポリシーが正しく設定されているか確認

### 問題3: 音声が再生されない

**原因**: ブラウザの音声再生がブロックされている

**解決方法**:
1. ブラウザのコンソールでエラーを確認
2. HTTPSでアクセスしているか確認（localhostは問題なし）
3. ブラウザの音声自動再生設定を確認

### 問題4: 台本生成が長すぎる/短すぎる

**原因**: 記事の数や長さが想定と異なる

**解決方法**:
1. `radioService.ts` の `targetDurationMinutes` パラメータを調整
2. プロンプトのテンプレートを調整して長さを制御

## 技術仕様

### 生成フロー

```
記事取得 → 優先度ソート → 台本生成 → セグメント分割 → TTS × 3 → 音声結合 → アップロード → DB保存
```

### 処理時間

- **記事取得**: 1-2秒
- **台本生成**: 5-10秒（Gemini 2.0 Flash）
- **TTS生成**: 10-20秒 × 3セグメント = 30-60秒
- **音声結合**: 3-5秒
- **アップロード**: 5-10秒
- **合計**: 約50-90秒

### ファイルサイズ

- **4分の音声**: 約5-10MB（WAV形式）
- **最大サイズ制限**: 50MB

### データ構造

#### `radio_programs` テーブル

| カラム名 | 型 | 説明 |
|---------|-----|------|
| `id` | UUID | ラジオ番組ID |
| `newsletter_id` | UUID | 元のデジタル回覧板ID |
| `title` | TEXT | 番組タイトル |
| `script` | TEXT | 2人の掛け合い台本 |
| `audio_url` | TEXT | 音声ファイルURL |
| `duration_seconds` | INTEGER | 長さ（秒） |
| `generation_status` | TEXT | ステータス |
| `article_count` | INTEGER | 使用した記事数 |

## よくある質問（FAQ）

### Q1: 何件の記事からラジオ番組を作れますか？

A: 最大5件の記事を使用します。優先度が高い記事から自動的に選択されます。

### Q2: 音声の声を変更できますか？

A: 現在は固定の音声（Gemini TTS: Kore）を使用しています。将来的に複数の音声を選択できるようにする予定です。

### Q3: BGMを追加できますか？

A: 現在は未対応ですが、将来的に著作権フリーのBGMを自動挿入する機能を追加予定です。

### Q4: ラジオ番組を削除するには？

A: 現在は手動で削除する必要があります。Supabase Dashboard > Table Editor > radio_programs からレコードを削除してください。

### Q5: 1つのデジタル回覧板に複数のラジオ番組を作れますか？

A: 現在は1つのデジタル回覧板につき1つのラジオ番組のみです。再生成すると上書きされます。
