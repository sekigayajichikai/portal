# ハイブリッド方式 使用ガイド

月別デジタル回覧板で、自治会の回覧板と外部団体（学校、駐在所など）の資料を統合管理する方法を説明します。

## 概要

### ハイブリッド方式とは

**2つの抽出モード**を使い分けることで、効率的にデジタル回覧板を作成できます：

| モード | 対象資料 | 処理内容 | 処理時間 | PDF添付 | 用途 |
|--------|----------|----------|----------|---------|------|
| **自治会の回覧板** | 自治会発行の広報誌 | 4段階要約（headline/brief/summary/content） | 30-60秒 | ✅ 元PDF添付 | 記事を詳しく作成 |
| **地域の知らせ** | 学校便り、駐在所お知らせなど | タイトル + 1行要約 | 5-10秒 | ✅ 元PDF添付 | PDFで保管（簡単登録） |

### メリット

1. **作業効率**: 外部資料は「地域の知らせ」で素早く追加
2. **情報の正確性**: すべての記事から元のPDFを閲覧できる
3. **統一された体験**: すべての資料を1つのデジタル回覧板で管理
4. **検索・フィルタリング**: カテゴリ分類で必要な情報を素早く見つけられる
5. **元資料の参照**: 記事を読みながらワンクリックで元PDFを確認可能

## セットアップ手順

### 1. Supabase Storageのセットアップ

PDFファイルを保存するためのストレージバケットを作成します。

#### 開発環境と本番環境の違い

| 項目 | 開発環境 | 本番環境 |
|------|----------|----------|
| **使用スクリプト** | `storage-setup-dev.sql` | `storage-setup-prod.sql` |
| **アップロード権限** | 匿名ユーザー（anon）+ 認証済み | 認証済みユーザーのみ |
| **認証機能** | 不要 | 必須（Supabase Auth） |
| **セキュリティ** | 低（テスト用） | 高（本番用） |
| **推奨環境** | テスト用Supabaseプロジェクト | 本番用Supabaseプロジェクト |

#### 開発環境のセットアップ

**推奨**: 動作確認を素早く行いたい場合

1. Supabaseダッシュボード (https://supabase.com) にログイン
2. **テスト用**プロジェクトを選択
3. 左メニューから「SQL Editor」を開く
4. 「New query」をクリック
5. `storage-setup-dev.sql` の内容をコピー＆ペースト
6. 「Run」ボタンをクリック

**⚠️ 重要**: 
- 開発環境用スクリプトは**誰でもPDFをアップロード可能**です
- テスト用のSupabaseプロジェクトでのみ使用してください
- 本番環境では絶対に使用しないでください

**確認方法:**
- 左メニューの「Storage」を開く
- `newsletters` バケットが作成されていることを確認
- すぐにアプリケーションでテスト可能

#### 本番環境のセットアップ

**推奨**: セキュアな本番環境を構築する場合

1. Supabaseダッシュボード (https://supabase.com) にログイン
2. **本番用**プロジェクトを選択
3. 左メニューから「SQL Editor」を開く
4. 「New query」をクリック
5. `storage-setup-prod.sql` の内容をコピー＆ペースト
6. 「Run」ボタンをクリック

**注意事項**:
- 認証済みユーザーのみがPDFをアップロード可能
- Supabase Authの実装が必要です（将来対応）
- 現時点では認証機能がないため、PDFアップロードは動作しません

**確認方法:**
- 左メニューの「Storage」を開く
- `newsletters` バケットが作成されていることを確認
- 認証機能を実装してからテスト

### 2. 環境変数の確認

`.env.local` ファイルに以下が設定されていることを確認：

```env
# Supabase
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# AI Provider (Anthropic or OpenRouter)
VITE_AI_PROVIDER=anthropic
VITE_ANTHROPIC_API_KEY=your_anthropic_api_key
```

### 3. 依存パッケージのインストール

storageServiceが使用するSupabaseクライアントは既にインストール済みです。追加のパッケージインストールは不要です。

## 使い方

### 基本フロー

```
1. デジタル回覧板を作成（例: 2025年1月号）
   ↓
2. 自治会の回覧板を「詳細抽出」で追加
   ↓
3. 学校便りを「簡易抽出」で追加
   ↓
4. 駐在所お知らせを「簡易抽出」で追加
   ↓
5. すべての記事を確認して保存
```

### 詳細な手順

#### Step 1: デジタル回覧板を作成

1. 管理画面の「デジタル回覧板」タブを開く
2. 年と月を選択（例: 2025年1月）
3. 「月号形式で作成」ボタンをクリック

または、カスタムタイトルを入力：
- 「○○自治会 1月号」など
- 「カスタムタイトルで作成」ボタンをクリック

#### Step 2: 自治会の回覧板を追加

1. **抽出モードを選択**:
   - 「自治会の回覧板」を選択（デフォルト）

2. **PDFをアップロード**:
   - 「クリックしてPDFを選択」をクリック
   - 自治会の回覧板PDFを選択

3. **記事を抽出**:
   - 「記事を抽出して追加」ボタンをクリック
   - 30-60秒待つ（AI処理中）

4. **メタデータ入力**:
   - PDFタイトル: 「関ヶ谷だより」（AI が自動提案）
   - 号数: 「第123号」（任意）
   - 「確定して記事を抽出」をクリック

5. **結果確認**:
   - 記事一覧に詳細な記事が追加される
   - 4段階要約（headline/brief/summary/content）が生成される
   - **全記事に元PDFが添付される**（記事を展開すると「添付ファイル」セクションが表示）

#### Step 3: 外部資料を追加

1. **抽出モードを変更**:
   - 「地域の知らせ」を選択

2. **PDFをアップロード**:
   - 「クリックしてPDFを選択」をクリック
   - 学校便りや駐在所お知らせのPDFを選択

3. **記事を抽出**:
   - 「記事を抽出して追加」ボタンをクリック
   - 5-10秒待つ（軽量処理）

4. **メタデータ入力**:
   - PDFタイトル: 「○○小学校だより」（AI が自動提案）
   - 号数: 「1月号」（任意）
   - 「確定して記事を抽出」をクリック

5. **結果確認**:
   - 記事一覧に簡易記事が追加される
   - タイトルと1行要約のみ（処理が高速）
   - **元PDFが添付される**（記事を展開すると「添付ファイル」セクションが表示）

#### Step 4: 記事の確認と保存

1. **記事一覧を確認**:
   - カテゴリフィルターで絞り込み
   - 詳細度を切り替え（見出し/簡潔/要約/全文）
   - 記事をクリックして展開

2. **PDF添付を確認**:
   - 簡易抽出で追加した記事を展開
   - 「添付ファイル」セクションが表示される
   - PDFリンクをクリックして内容を確認

3. **記事を編集**（必要に応じて）:
   - 記事を展開して「記事を編集」ボタンをクリック
   - タイトル、カテゴリ、優先度などを修正

4. **保存**:
   - 「記事を保存」ボタンをクリック
   - Supabaseに保存される

## 実装の詳細

### 処理フロー（共通）

**すべてのモードで元PDFをStorageにアップロードします**

```
1. PDFファイルを選択
   ↓
2. Supabase Storageにアップロード
   ├─ バケット: newsletters
   ├─ パス: YYYY/MM/prefix-filename-timestamp.pdf
   ├─ プレフィックス: 
   │  ├─ 詳細抽出: "source-"
   │  └─ 簡易抽出: "attachment-"
   └─ 公開URL取得
   ↓
3. Claude APIで記事抽出
   ├─ 詳細抽出: 4段階要約 × 複数記事
   └─ 簡易抽出: タイトル + 1行要約 × 1記事
   ↓
4. すべての記事にPDFを添付
   └─ attachments: [{ type: 'pdf', url: '...', filename: '...' }]
   ↓
5. accumulatedArticlesに追加
```

### 2つのモードの違い

| 項目 | 自治会の回覧板 | 地域の知らせ |
|------|----------|----------|
| **記事数** | 複数（PDFの内容に応じて） | 1記事のみ |
| **要約レベル** | 4段階（headline/brief/summary/content） | 1行要約のみ |
| **PDF添付** | ✅ 全記事に同じPDFを添付 | ✅ 記事に添付 |
| **処理時間** | 30-60秒 | 5-10秒 |
| **用途** | 記事を詳しく作成 | PDFで保管（簡単登録） |

### データ構造

#### 自治会の回覧板（記事を詳しく作成）

```typescript
{
  id: "a-1234567890-0",
  newsletter_id: "n-1234567890",
  organization_id: "org1",
  
  // 基本情報
  title: "どんど焼き開催のお知らせ",
  category: "event",
  priority: "high",
  deadline: "2026-01-10",
  
  // 4段階要約
  headline: "どんど焼き",
  brief: "1/10どんど焼き開催",
  summary: "1月10日（土）11時～奥座公園でどんど焼き。お焚き上げ、豚汁振る舞い、餅つき体験",
  content: "## イベント概要\n日時: **1月10日（土）10:00-12:00**...",
  
  // メタ情報
  tags: ["正月", "イベント", "奥座公園"],
  visibility: "public",
  source: "関ヶ谷だより 第123号",
  // 🆕 元PDFを添付（全記事共通）
  attachments: [
    {
      type: "pdf",
      url: "https://xxx.supabase.co/storage/v1/object/public/newsletters/2025/01/source-sekigaya-tayori-1234567890.pdf",
      filename: "sekigaya-tayori.pdf"
    }
  ],
  
  created_at: "2026-01-21T10:00:00Z",
  updated_at: "2026-01-21T10:00:00Z"
}
```

#### 地域の知らせ（PDFで保管）

```typescript
{
  id: "a-1234567891-0",
  newsletter_id: "n-1234567890",
  organization_id: "org1",
  
  // 基本情報
  title: "○○小学校だより 1月号",
  category: "announcement",
  priority: "medium",
  deadline: null,
  
  // 最小限の要約
  headline: "学校便り",
  brief: "1月の行事予定表",
  summary: "1月の行事予定表",
  content: "詳細は添付のPDFファイルをご覧ください。\n\n1月の行事予定表",
  
  // メタ情報
  tags: ["学校", "行事"],
  visibility: "public",
  source: "○○小学校だより 1月号",
  attachments: [
    {
      type: "pdf",
      url: "https://xxx.supabase.co/storage/v1/object/public/newsletters/2025/01/attachment-school-newsletter-1234567890.pdf",
      filename: "school-newsletter.pdf"
    }
  ],
  
  created_at: "2026-01-21T10:05:00Z",
  updated_at: "2026-01-21T10:05:00Z"
}
```

### ストレージ構造

```
newsletters/
├─ 2025/
│  ├─ 01/
│  │  ├─ source-sekigaya-tayori-1705819100000.pdf      # 詳細抽出（自治会回覧板）
│  │  ├─ attachment-school-newsletter-1705819200000.pdf # 簡易抽出（学校便り）
│  │  ├─ attachment-police-notice-1705819250000.pdf     # 簡易抽出（駐在所）
│  │  └─ attachment-community-center-1705819300000.pdf  # 簡易抽出（地区センター）
│  ├─ 02/
│  │  └─ ...
│  └─ ...
└─ 2026/
   └─ ...
```

**ファイル名のプレフィックス**:
- `source-`: 自治会の回覧板（記事を詳しく作成）
- `attachment-`: 地域の知らせ（PDFで保管）

## トラブルシューティング

### PDFアップロードに失敗する（`new row violates row-level security policy`）

**原因**: Supabase Storageのセキュリティポリシーが認証済みユーザーのみを許可している

**対処法**:
1. **開発環境の場合**: `storage-setup-dev.sql` を実行してください
   - 既存のポリシーを削除してから実行します
   - スクリプトに `DROP POLICY IF EXISTS` が含まれているため安全です
2. **本番環境の場合**: 認証機能を実装してから `storage-setup-prod.sql` を使用してください

### ポリシーが既に存在するエラー

**エラー**: `policy "Public Access" for table "objects" already exists`

**対処法**:
- 更新版のスクリプト（`storage-setup-dev.sql` または `storage-setup-prod.sql`）を使用してください
- スクリプトには既存ポリシーを削除するコードが含まれています
- 再実行しても問題ありません

### Supabase Storageが未設定

**対処法**:
1. 環境に応じたスクリプトを実行したか確認
   - 開発環境: `storage-setup-dev.sql`
   - 本番環境: `storage-setup-prod.sql`
2. Supabaseダッシュボードで `newsletters` バケットが存在するか確認
3. `.env.local` に `VITE_SUPABASE_URL` と `VITE_SUPABASE_ANON_KEY` が設定されているか確認

### 地域の知らせの登録が遅い

**原因**: ネットワークまたはAPI遅延

**対処法**:
- PDFファイルサイズを確認（推奨: 5MB以下）
- Claude APIキーが正しく設定されているか確認
- ブラウザのコンソールでエラーを確認

### PDF添付が表示されない

**原因**: attachmentsフィールドが空

**対処法**:
1. 記事を展開して確認
2. どちらのモードで追加したか確認（両モードともPDF添付あり）
3. ブラウザのコンソールでエラーを確認

## よくある質問

### Q1. すべてのPDFを「地域の知らせ」で追加できますか？

**A**: はい、可能です。ただし、自治会の回覧板は「自治会の回覧板」モードを推奨します。理由：
- 4段階要約で詳細な情報を提供できる
- 見出し・簡潔・要約での切り替えが可能
- 検索性が高い

### Q2. 後からモードを変更できますか？

**A**: 現時点では、一度追加した記事のモードは変更できません。削除して再追加する必要があります。

### Q3. PDFファイルのサイズ制限はありますか？

**A**: Supabaseのデフォルト制限は50MBです。ただし、以下を推奨：
- 1ファイルあたり5MB以下
- 画質を調整してファイルサイズを削減

### Q4. 無料プランでどれくらいのPDFを保存できますか？

**A**: Supabaseの無料プランは1GBです。目安：
- 1PDFあたり2MBと仮定
- 約500個のPDFを保存可能
- 月に10個追加しても約4年間使用可能

### Q5. 古いPDFは自動削除されますか？

**A**: いいえ、自動削除されません。手動で削除する必要があります。将来的に自動アーカイブ機能を追加予定です。
