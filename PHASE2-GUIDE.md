# Phase 2 UX改善機能 使用ガイド

Phase 2で追加されたUX改善機能の使い方を説明します。

## 新機能概要

### 1. PDFタイトル/号数入力（AI提案付き）
- PDFアップロード時にAIがタイトルと号数を自動提案
- ユーザーが確認・編集可能

### 2. 登録済みPDF個別削除
- 登録済みPDFを個別に削除可能
- 削除時、そのPDFから抽出された記事も一緒に削除

### 3. 記事のソース表示
- 各記事にソース情報（どのPDFから抽出されたか）を表示
- 一目でどの広報誌から来た記事かわかる

### 4. 記事の並び替え
- 優先度順、カテゴリ順、ソース順、日付順で並び替え可能
- フィルターと組み合わせて使用可能

## 使用方法

### 機能1: PDFタイトル/号数入力（AI提案付き）

#### ステップ1: Newsletterを作成

```
1. 「PDF広報誌」タブをクリック
2. タイトル入力（例: 2025年1月号）
3. 「Newsletterを作成」ボタンをクリック
```

#### ステップ2: PDFを選択

```
1. 「PDFを追加」セクションで「クリックしてPDFを選択」
2. PDFファイルを選択（例: sekigaya-dayori.pdf）
3. 「記事を抽出して追加」ボタンをクリック
```

#### ステップ3: AIが提案したメタデータを確認

**PDFメタデータ入力ダイアログが表示されます：**

```
┌─────────────────────────────────┐
│ PDF情報を入力            [×]    │
├─────────────────────────────────┤
│ ファイル名                      │
│ sekigaya-dayori.pdf             │
│                                 │
│ ✨ タイトル（AI提案）           │
│ [関ヶ谷だより              ]   │
│                                 │
│ ✨ 号数（AI提案）               │
│ [第123号                   ]   │
│                                 │
│ [キャンセル] [確定して記事を抽出]│
└─────────────────────────────────┘
```

**AIの処理:**
- PDFの先頭部分を自動解析
- タイトルと号数を自動抽出
- 数秒で提案を表示

#### ステップ4: 確認・編集・確定

1. AIが提案したタイトルを確認
   - 正しければそのまま
   - 修正が必要なら編集
2. 号数も同様に確認・編集
3. 「確定して記事を抽出」ボタンをクリック
4. 20-60秒待つ（記事抽出中）
5. 記事が追加される

**結果:**
```
✅ 関ヶ谷だよりから3件の記事を追加しました
   処理時間: 25.3秒
   合計: 3件の記事
```

### 機能2: 登録済みPDF個別削除

#### 削除手順

```
1. 登録済みPDF一覧を確認
2. 削除したいPDFの右側にある[🗑]ボタンをクリック
3. 確認ダイアログで「OK」をクリック
4. PDFとその記事が削除される
```

**削除前:**
```
┌────────────────────────────────────┐
│ 登録済みPDF (2件)                  │
│ ┌────────────────────────────────┐ │
│ │ 📄 関ヶ谷だより 第123号        │ │
│ │    3件の記事 • sekigaya.pdf    │ │
│ │    14:23   [🗑 削除]           │ │
│ └────────────────────────────────┘ │
│ ┌────────────────────────────────┐ │
│ │ 📄 会報ふれあい 2025年1月号    │ │
│ │ │    5件の記事 • fureai.pdf      │ │
│ │    14:28   [🗑 削除]           │ │
│ └────────────────────────────────┘ │
└────────────────────────────────────┘
```

**削除ダイアログ:**
```
「関ヶ谷だより」を削除しますか？
このPDFから抽出された記事も削除されます。

[キャンセル] [OK]
```

**削除後:**
```
┌────────────────────────────────────┐
│ 登録済みPDF (1件)                  │
│ ┌────────────────────────────────┐ │
│ │ 📄 会報ふれあい 2025年1月号    │ │
│ │    5件の記事 • fureai.pdf      │ │
│ │    14:28   [🗑 削除]           │ │
│ └────────────────────────────────┘ │
└────────────────────────────────────┘

✅ 「関ヶ谷だより」とその記事を削除しました
```

**注意事項:**
- 削除は取り消せません
- そのPDFから抽出された記事もすべて削除されます
- 記事数は自動的に再計算されます

### 機能3: 記事のソース表示

#### ソース情報の確認

記事カードに自動的にソース情報が表示されます：

```
┌────────────────────────────────────┐
│ ■ どんど焼き開催のお知らせ        │
│   [重要] [イベント] [一般公開]    │
│   [📄 関ヶ谷だより 第123号]       │← ソース表示
│                                    │
│   1月15日（日）に地域でどんど焼き...│
└────────────────────────────────────┘
```

**表示される情報:**
- PDFのタイトル
- 号数（設定されている場合）

**利点:**
- どの広報誌から来た記事かすぐわかる
- 複数PDF統合時に情報の出所が明確
- トレーサビリティの向上

### 機能4: 記事の並び替え

#### ソート機能の使い方

記事一覧の上部にある「並び順」で選択：

```
┌────────────────────────────────────┐
│ カテゴリ: [全て] [お知らせ] [イベント]│
│ 並び順: [デフォルト ▼]            │← ここで選択
│ 詳細度: [見出し][簡潔][要約][全文]│
└────────────────────────────────────┘
```

#### ソートオプション

**1. デフォルト**
- 追加された順番（PDF追加順）
- 特別な並び替えなし

**2. 優先度順**
```
高（重要）
├─ どんど焼き開催のお知らせ
├─ 民生委員災害時支援アンケート
中（確認推奨）
├─ 会館の臨時休館について
└─ 資源ごみ収集日の変更
低（参考情報）
└─ 図書部新着本のお知らせ
```

**3. カテゴリ順**
```
イベント
├─ どんど焼き開催のお知らせ
├─ 新春俳句会の報告
お知らせ
├─ 会館の臨時休館について
└─ 資源ごみ収集日の変更
```

**4. ソース順**
```
会報ふれあい 2025年1月号
├─ 新春俳句会の報告
├─ 資源ごみ収集日の変更
├─ 防災訓練のお知らせ
関ヶ谷だより 第123号
├─ どんど焼き開催のお知らせ
├─ 会館の臨時休館について
```

**5. 日付順（新しい順）**
```
2025-01-20 14:28 防災訓練のお知らせ
2025-01-20 14:23 会館の臨時休館について
2025-01-20 14:23 どんど焼き開催のお知らせ
```

#### フィルターとの組み合わせ

```
【例1】イベントカテゴリ × 優先度順
→ イベントの中で重要なものから表示

【例2】すべて × ソース順
→ 広報誌ごとにまとめて表示

【例3】お知らせ × 日付順
→ 最新のお知らせから表示
```

## トラブルシューティング

### Q1. AIのタイトル提案が正しくない

**原因:** PDFのレイアウトが特殊、またはタイトル情報が不明瞭

**対処:**
1. 提案されたタイトルを手動で修正
2. 正しいタイトルを入力して確定

### Q2. メタデータダイアログが表示されない

**原因:** APIキーが設定されていない、またはPDFの読み込みエラー

**対処:**
1. `.env`ファイルで`VITE_ANTHROPIC_API_KEY`を確認
2. サーバーを再起動（Ctrl+C → npm run dev）
3. PDFファイルサイズを確認（10MB以下）

### Q3. PDFを削除したのに記事数が合わない

**原因:** ブラウザキャッシュの問題

**対処:**
1. ページをリロード（F5）
2. リセットボタンで編集をやり直し

### Q4. ソート機能が効かない

**原因:** カテゴリフィルターとの組み合わせによる

**対処:**
1. カテゴリフィルターを「すべて」に戻す
2. ソートを再選択
3. 記事数を確認

## よくある質問

### Q1. AIのタイトル提案はどこまで正確ですか？

約80-90%の精度です。特に：
- 日本語の広報誌に最適化
- 先頭ページのヘッダー/タイトルを重点的に解析
- 号数表記（第○号、○○年○月号）に対応

### Q2. タイトル/号数は後から変更できますか？

現在はPDFを削除して再アップロードする必要があります。
Phase 3で編集機能を追加予定です。

### Q3. ソース情報は記事に永続的に保存されますか？

はい。記事の`source`フィールドに保存されます。
Supabase統合後もデータベースに保存されます。

### Q4. 複数のソートを組み合わせられますか？

現在は1つのソート条件のみです。
Phase 3で複数条件ソートを検討中です。

### Q5. PDFのタイトルを空にできますか？

いいえ。タイトルは必須です。
最低でも「広報誌」などの汎用名を入力してください。
